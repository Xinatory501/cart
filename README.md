# CartaMe Bot — Версия 1.4 (Новое Обновление)

Телеграм-бот поддержки CartaMe с запуском от одного до шести ботов:

- `BOT1` - Беларусь
- `BOT2` - Казахстан
- `BOT3` - Узбекистан
- `BOT4` - дополнительный бот
- `BOT5` - дополнительный бот
- `BOT6` - дополнительный бот

Все боты работают на единой кодовой базе и базе данных, но у каждого свой токен, свой профиль и свои топики в группе поддержки.

---

## 🚀 Что нового в версии 1.4

### 1. Цифровые номера обращений (тикеты)
- Формат 6-значных номеров тикетов переведен со смешанного буквенно-цифрового формата на **строго цифровой** (например, `481516` или `104859`). Это упрощает передачу номеров поддержке и их поиск.
- Валидация в админ-панели полностью обновлена для проверки 6-значных цифровых номеров.

### 2. Динамические клавиатуры (Reply Keyboard)
- Улучшено юзабилити за счет динамического переключения физических кнопок внизу экрана:
  * **Вне активного чата**: Отображается аккуратная кнопка `start` для быстрого перехода в главное меню.
  * **В активном чате**: Клавиатура переключается на кнопку `Узнать номер диалога` (кнопка переведена на все поддерживаемые языки: Русский, Английский, Узбекский, Казахский). При нажатии на нее пользователь мгновенно получает свой активный ID обращения без засорения диалога с AI или поддержкой.

### 3. Приветственные стикеры в админ-панели
- В панель администратора добавлен раздел **«🖼️ Приветственный стикер»**.
- Администратор может отправить боту любой стикер, и он сохранится в базе данных как приветственный (или сбросить его обратно в стандартный эмодзи `👋`).
- Настроен правильный порядок вывода: стикер с прикрепленной к нему клавиатурой `start` отправляется в чат первым, а уже после него — приветственный баннер и текст.

### 4. Исправление привязки ботов (Auto-Assignment)
- Устранена критическая проблема с недоставкой сообщений от техподдержки: теперь при получении сообщений в топике группа автоматически привязывает пользователя к активному запущенному боту на лету.

### 5. Премиум дизайн экспорта чатов
- Экспорт диалогов в **PDF** получил улучшенный визуал: пузыри сообщений имеют закругленные углы, а роли участников разделены по цветам (Синий для собеседника, Зеленый для AI-ассистента, Сиреневый для техподдержки).
- Персональные данные пользователя анонимизируются: в выгрузках (TXT и PDF) имя пользователя везде заменяется на **«Собеседник»**.
- Реализовано четкое разделение сессий переписок с границами начала и окончания тикетов.

### 6. HTTP API интеграция
- Внедрен HTTP API сервер (`GET /api/chat/<user_id>`), позволяющий внешним системам запрашивать полную историю переписок пользователя в JSON-формате.

### 7. Очистка репозитория
- Из проекта удалены все лишние каталоги и файлы, не относящиеся к работе бота (например, папка `android_flash`).

## 🛠️ Основной функционал бота

1. **Мультиботность (Multi-bot)**: Запуск до 6 независимых ботов на одной БД. Каждый бот ведет диалог на своем языке по умолчанию и создает свои топики.
2. **AI-Ассистент (Groq/OpenAI)**: Бот автоматически отвечает на вопросы пользователей, используя базу знаний и обучающие примеры.
3. **Бесшовная передача оператору**: Бот умеет детектировать запросы переключения на оператора (`call_people` / "Позвать человека"), при этом AI отключается, а в группе поддержки создается отдельный топик для прямого общения с администратором.
4. **Админ-панель (`/admin`)**:
   * Настройка API ключей и провайдеров.
   * Конфигурация группы поддержки.
   * Установка лимитов антифлуда.
   * Настройка политики конфиденциальности и обучение AI.
   * Управление пользователями (просмотр инфо, бан, выдача прав).
   * Экспорт переписок по номеру тикета.
5. **Безопасность**: Защита от спама (Anti-Flood) и система блокировок пользователей.

## 📦 Подготовка к запуску

### 1. Подготовка `.env`
Создайте файл `.env` в корне проекта и заполните его:

```env
ADMIN_IDS=123456789
BOT1_TOKEN=токен_первого_бота
BOT2_TOKEN=
BOT3_TOKEN=
BOT4_TOKEN=
BOT5_TOKEN=
BOT6_TOKEN=
SUPPORT_GROUP_ID=-1001234567890
```

*Без `BOT1_TOKEN` приложение не запустится. Остальные токены необязательны.*

### 2. Запуск через Docker (Рекомендуется)

```bash
docker compose up -d --build
```

Остановка:
```bash
docker compose down
```

Логи:
```bash
docker compose logs -f
```

### 3. Локальный запуск без Docker

1. Установите Python 3.11.
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Linux/macOS
   .venv\Scripts\activate     # Для Windows
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите бота:
   ```bash
   python bot.py
   ```
