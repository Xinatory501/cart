# CartaMe Bot

Телеграм-бот поддержки CartaMe с запуском от одного до шести ботов:

- `BOT1` - Беларусь
- `BOT2` - Казахстан
- `BOT3` - Узбекистан
- `BOT4` - дополнительный бот
- `BOT5` - дополнительный бот
- `BOT6` - дополнительный бот

Все боты работают на одном коде, но у каждого свой токен, свой профиль и свои топики в группе поддержки.

## Что нужно перед запуском

1. От одного до шести Telegram-ботов, созданных через `@BotFather`.
2. Минимально нужен токен первого бота:
   - `BOT1_TOKEN`
3. Дополнительные токены можно подключать по мере необходимости:
   - `BOT2_TOKEN`
   - `BOT3_TOKEN`
   - `BOT4_TOKEN`
   - `BOT5_TOKEN`
   - `BOT6_TOKEN`
4. Telegram-группа поддержки с включенными темами.
5. Все используемые боты должны быть добавлены в эту группу.
6. У всех используемых ботов в группе должны быть права администратора:
   - отправка сообщений
   - управление темами
7. Telegram ID администратора, который будет управлять ботом через `/admin`.
8. Docker Desktop или Python 3.11 для локального запуска.

## Подготовка `.env`

Создайте файл `.env` в корне проекта и заполните его.

Пример:

```env
ADMIN_IDS=123456789
BOT1_TOKEN=token_for_belarus_bot
BOT2_TOKEN=
BOT3_TOKEN=
BOT4_TOKEN=
BOT5_TOKEN=
BOT6_TOKEN=
SUPPORT_GROUP_ID=-1001234567890
```

Пояснения:

- `ADMIN_IDS` - Telegram ID админа или список ID через запятую.
- `BOT1_TOKEN` - бот для Беларуси.
- `BOT2_TOKEN` - бот для Казахстана.
- `BOT3_TOKEN` - бот для Узбекистана.
- `BOT4_TOKEN` - дополнительный бот.
- `BOT5_TOKEN` - дополнительный бот.
- `BOT6_TOKEN` - дополнительный бот.
- `SUPPORT_GROUP_ID` - ID группы поддержки.

Важно:

- Без `BOT1_TOKEN` бот не запустится.
- `BOT2_TOKEN`-`BOT6_TOKEN` можно оставить пустыми.
- Если `SUPPORT_GROUP_ID` не указан, не будут работать топики поддержки и логов.

## Как узнать `SUPPORT_GROUP_ID`

Самый простой способ:

1. Добавьте бота в нужную группу.
2. Включите в группе темы.
3. Отправьте любое сообщение в одной из тем.
4. Скопируйте ссылку на сообщение.

Ссылка будет вида:

```text
https://t.me/c/1234567890/15
```

Из нее получается ID группы:

```text
-1001234567890
```

## Запуск через Docker

Основной вариант запуска:

```powershell
docker compose up -d --build
```

Остановка:

```powershell
docker compose down
```

Просмотр логов:

```powershell
docker compose logs -f
```

## Локальный запуск без Docker

Подходит для отладки.

1. Установите Python 3.11.
2. Создайте виртуальное окружение:

```powershell
python -m venv .venv
```

3. Активируйте его:

```powershell
.venv\Scripts\activate
```

4. Установите зависимости:

```powershell
pip install -r requirements.txt
```

5. Запустите бота:

```powershell
python bot.py
```

## Что происходит при первом запуске

При первом старте приложение:

- создаст базу данных в `data/cartame_bot.db`
- создаст таблицы
- добавит стартовые настройки
- создаст дефолтных AI-провайдеров в базе
- отключит тестовые placeholder-ключи
- назначит администраторов из `ADMIN_IDS`
- для каждого запущенного бота создаст свой топик `Логи`, если настроена группа поддержки

## Настройка после запуска

После старта зайдите в нужного бота и выполните:

```text
/admin
```

Дальше в админке:

1. Проверьте группу поддержки.
2. Добавьте рабочий AI-ключ.
3. Проверьте модель.
4. При необходимости настройте антифлуд и политику конфиденциальности.

Важно:

- Пока не добавлен рабочий AI-ключ, бот не сможет отвечать пользователям.
- Ошибки пишутся не пользователю, а в отдельный топик `Логи`.

## Как работает поддержка в группе

- У каждого пользователя создается отдельный топик поддержки.
- Каждый бот работает только со своими пользователями и своими топиками.
- Сообщения из группы не должны блокироваться антифлудом.
- Если происходит ошибка, пользователь получает короткое сообщение вида "Произошла ошибка. Попробуйте позже.", а подробности уходят в топик `Логи`.

## Проверка, что все запустилось правильно

Проверьте по шагам:

1. Контейнер поднялся без падений.
2. В логах нет ошибки про отсутствие токенов.
3. Все подключенные боты отвечают на `/start`.
4. В группе поддержки появились топики `Логи`.
5. После сообщения пользователю создается отдельный топик поддержки.
6. Команда `/admin` открывается у администратора.

## Частые проблемы

### Бот не стартует

Проверьте:

- заполнен ли `BOT1_TOKEN`
- если используются дополнительные боты, заполнены ли `BOT2_TOKEN`-`BOT6_TOKEN`
- нет ли лишних пробелов в `.env`
- действительно ли токены рабочие

### Бот не пишет в группу поддержки

Проверьте:

- указан ли `SUPPORT_GROUP_ID`
- включены ли темы в группе
- выданы ли права на управление темами
- добавлены ли в группу все боты, которые реально запускаются

### Пользователь пишет, а AI не отвечает

Проверьте:

- добавлен ли рабочий AI-ключ через `/admin`
- активна ли модель
- нет ли ошибок в топике `Логи`

### Бот отвечает не в своем топике

Проверьте:

- что пользователь начал диалог именно с нужным ботом
- что после запуска не было ошибок в топике `Логи`
- что все запущенные боты обновлены и перезапущены

## Рекомендуемый порядок запуска с нуля

1. Создать от одного до шести ботов в `@BotFather`.
2. Создать группу поддержки.
3. Включить темы в группе.
4. Добавить туда все используемые боты.
5. Выдать им права администратора и управление темами.
6. Заполнить `.env`.
7. Выполнить `docker compose up -d --build`.
8. Зайти под админом в любого из ботов.
9. Выполнить `/admin`.
10. Добавить рабочий AI-ключ и проверить модель.
11. Протестировать `/start` у всех подключенных ботов.
