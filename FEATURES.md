# Новые функции CartaMe Bot

## 1. Система уточняющих вопросов от AI

### Описание
AI теперь может задавать уточняющие вопросы пользователю, когда запрос неоднозначный.

### Как это работает

**Пример 1: Неоднозначный вопрос о картах**
```
Пользователь: "как добавить карту"
AI: "Вы хотите добавить банковскую карту для оплаты или карту лояльности магазина?"
Пользователь: "карту лояльности"
AI: [дает инструкцию по добавлению карты лояльности]
```

**Пример 2: Неясная проблема**
```
Пользователь: "не работает"
AI: "Что именно не работает? Не открывается приложение, не сканируется QR-код или что-то другое?"
Пользователь: "не сканируется QR"
AI: [дает решение проблемы со сканированием]
```

### Технические детали

- **Токен активации:** `need_clarification`
- **Состояние:** `UserStates.waiting_clarification`
- **Таблица БД:** `clarification_contexts`

Когда AI добавляет токен `need_clarification` в ответ, система:
1. Сохраняет контекст (оригинальный вопрос + уточняющий вопрос)
2. Переводит пользователя в состояние ожидания ответа
3. После ответа объединяет вопросы и отправляет AI для финального ответа

### Настройка через Training Messages

Добавьте обучающие сообщения через `/admin` -> Training Messages:

```
Если вопрос неоднозначный, задай уточняющий вопрос:

Примеры:
- "как добавить карту" → спроси какую карту (банковскую или лояльности)
- "не работает" → спроси что именно не работает
- "ошибка" → попроси описать ошибку подробнее

После уточняющего вопроса добавь токен need_clarification
```

## 2. Улучшенная система Training Messages

### Описание
Полностью переработанный интерфейс управления обучающими сообщениями с inline-кнопками.

### Интерфейс

**Главное меню Training Messages:**
- Все обучающие сообщения отображаются как inline-кнопки
- Каждая кнопка показывает: статус (✅/❌), приоритет (⭐), превью текста
- Кнопки "➕ Добавить сообщение" и "◀️ Назад" внизу

**Детальный просмотр сообщения:**
При нажатии на сообщение показывается:
- Полный текст промпта
- Текущий статус (активно/неактивно)
- Приоритет (1-5)
- Объяснение системы приоритетов

Доступные действия:
- **✏️ Изменить содержание** - редактирование текста
- **🔢 Изменить приоритет** - выбор от 1 до 5
- **🔄 Вкл/Выкл** - активация/деактивация
- **🗑 Удалить** - удаление с подтверждением
- **◀️ Назад** - возврат к списку

### Система приоритетов

```
1 ⭐ - Слабая подсказка
AI может игнорировать при конфликте с другими инструкциями

2 ⭐⭐ - Обычная инструкция (по умолчанию)
AI следует в большинстве случаев

3 ⭐⭐⭐ - Важная инструкция
AI следует всегда, кроме критических ситуаций

4 ⭐⭐⭐⭐ - Очень важная инструкция
AI следует строго, имеет приоритет над обычными правилами

5 ⭐⭐⭐⭐⭐ - Критически важная инструкция
AI ВСЕГДА следует, наивысший приоритет
```

### Примеры использования

**Добавление нового сообщения:**
1. `/admin` -> Training Messages
2. Нажать "➕ Добавить сообщение"
3. Отправить текст инструкции
4. Сообщение добавляется с приоритетом 2

**Изменение приоритета:**
1. Выбрать сообщение из списка (нажать на кнопку)
2. Нажать "🔢 Изменить приоритет"
3. Выбрать нужный уровень (1-5)

**Редактирование содержания:**
1. Выбрать сообщение (нажать на кнопку)
2. Нажать "✏️ Изменить содержание"
3. Отправить новый текст

## 3. Загрузка и скачивание базы данных

### Описание
Возможность создавать резервные копии базы данных и восстанавливать из них.

### Функции

**📥 Скачать бекап:**
- Создает копию текущей базы данных
- Отправляет файл .db в чат
- Можно сохранить локально

**📤 Загрузить бекап:**
- Восстанавливает базу данных из файла
- Автоматически создает резервную копию текущей БД
- Требует подтверждения перед восстановлением

### Использование

**Создание бекапа:**
1. `/admin` -> Database Backup
2. Нажать "📥 Скачать бекап"
3. Дождаться создания и получить файл
4. Сохранить файл в безопасном месте

**Восстановление из бекапа:**
1. `/admin` -> Database Backup
2. Нажать "📤 Загрузить бекап"
3. Отправить файл .db
4. Проверить информацию о файле
5. Подтвердить восстановление
6. Текущая БД автоматически сохраняется как резервная копия

### Безопасность

- Перед восстановлением создается автоматический бекап текущей БД
- Резервная копия сохраняется как `cartame_bot.db.backup_before_restore`
- Все операции логируются в топик "Логи"
- Требуется подтверждение перед восстановлением

### Важно

⚠️ **При восстановлении:**
- Текущая база данных будет полностью заменена
- Все текущие данные будут потеряны (но сохранены в резервной копии)
- Бот продолжит работу с восстановленной БД без перезапуска

## Миграция базы данных

При первом запуске с новыми функциями автоматически создаются таблицы:

```sql
-- Контексты уточняющих вопросов
CREATE TABLE clarification_contexts (
    id INTEGER PRIMARY KEY,
    user_id BIGINT,
    session_id INTEGER,
    original_question TEXT,
    clarification_question TEXT,
    options TEXT,
    is_active BOOLEAN,
    created_at DATETIME,
    answered_at DATETIME
);
```

## Рекомендации по настройке

### Для уточняющих вопросов

1. Добавьте training message с приоритетом 3-4:
```
При неоднозначных вопросах задавай уточняющие вопросы.
Примеры неоднозначности:
- "карта" может означать банковскую или лояльности
- "не работает" - нужны детали
- "ошибка" - нужно описание

После уточняющего вопроса добавь: need_clarification
```

2. Добавьте конкретные примеры с приоритетом 2:
```
Примеры уточнений:
"как добавить карту" → "Какую карту - банковскую или дисконтную?"
"не работает приложение" → "Что именно не работает - не открывается, зависает или выдает ошибку?"
```

### Для системы приоритетов

- **Приоритет 5** - только для критических правил (безопасность, юридические требования)
- **Приоритет 4** - для важных бизнес-правил
- **Приоритет 3** - для стандартных инструкций
- **Приоритет 2** - для рекомендаций и примеров
- **Приоритет 1** - для мягких подсказок

### Для резервного копирования

- Создавайте бекапы регулярно (раз в день/неделю)
- Храните несколько версий бекапов
- Проверяйте возможность восстановления на тестовом окружении
- Логи восстановления доступны в топике "Логи"

## Тестирование

### Тест уточняющих вопросов

1. Запустите бота
2. Начните диалог: `/start`
3. Отправьте: "как добавить карту"
4. AI должен спросить уточнение
5. Ответьте: "карту лояльности"
6. AI должен дать конкретный ответ

### Тест Training Messages

1. `/admin` -> Training Messages
2. Добавьте тестовое сообщение
3. Нажмите на кнопку сообщения для просмотра
4. Проверьте изменение приоритета
5. Проверьте редактирование
6. Проверьте активацию/деактивацию
7. Удалите тестовое сообщение

### Тест резервного копирования

1. `/admin` -> Database Backup
2. Скачайте текущий бекап
3. Загрузите тот же файл обратно
4. Проверьте, что данные не изменились
5. Проверьте наличие резервной копии

## Логирование

Все действия логируются:
- Создание уточняющих контекстов
- Изменения training messages
- Изменения приоритетов
- Активация/деактивация сообщений
- Создание и восстановление бекапов

Логи доступны в топике "Логи" группы поддержки.
